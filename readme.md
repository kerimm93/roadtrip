# 🚗 Roadtrip

**Roadtrip** is a prompt-first development OS for software projects.

It helps you build software with AI **without drowning in context switching, feature sprawl, vague prompts, or unstructured iteration**.

Roadtrip is not just a feature tracker.
It is a structured layer between:

* ideas
* projects
* features
* code analysis
* sprint execution
* AI workflows
* learning loops
* long-term architecture decisions

At its core, Roadtrip exists to help developers **build, understand, review, and improve software iteratively** — instead of getting stuck in chaotic vibe-coding.

---

## ✨ Philosophy

Roadtrip follows one core principle:

> AI should accelerate learning — not replace understanding.

Many AI-assisted workflows stop at:

```text
Prompt → Code → Next Prompt → More Code
```

Roadtrip is designed for a more deliberate loop:

```text
Idea
→ Structure
→ Sprint
→ Build
→ Analyze
→ Review
→ Learn
→ Build something slightly more complex
```

The goal is not blind automation.
The goal is **controlled progression**.

Roadtrip helps developers:

* reduce cognitive overload
* separate signal from noise
* preserve project context
* avoid losing work in AI-chat chaos
* review changes consciously
* extract learnings from implementation
* improve incrementally
* rely on AI as scaffolding, not dependency

In short:

> Roadtrip helps you grow from AI-assisted building toward deeper technical understanding.

---

## 🚀 What Roadtrip Is Comparable To

Roadtrip overlaps partially with several tools, but is not identical to any of them.

It combines ideas from:

* **Jira / Linear** → feature and sprint tracking
* **Obsidian / lightweight PKM** → project notes and context
* **GitHub Projects / changelog thinking** → version-oriented thinking
* **Prompt orchestration tools** → AI workflow routing
* **Learning systems** → reflection and skill growth

Roadtrip acts as a **meta-layer above active software projects**.

## 🧠 Core Capabilities

### 📦 Project Management

Manage multiple software projects in parallel.

Projects can store:

* summary
* status
* current focus
* next step
* tags
* linked chats
* linked notes
* architecture hints
* repository links
* external references

---

### 🧩 Feature Intelligence

Roadtrip separates:

* **Planned features** → roadmap / desired state
* **Implemented features** → actual detected or confirmed state

This enables a real **Soll/Ist comparison** between intention and reality.

---

### 🤖 Prompt-First AI Workflows

Roadtrip is deliberately **prompt-first**.

It generates structured prompts for:

* sprint starts
* sprint handoffs
* code analysis
* cleanup review
* project momentum
* architecture planning
* learning extraction

AI proposes.
Roadtrip structures.
The user decides.

---

### 🔁 Sprint & Chat Routing

Roadtrip manages:

* project main chats
* sprint chats
* execution chats
* branch contexts
* handoff cycles

Core loop:

```text
Roadtrip → Main Chat → Roadtrip → Sprint Chat → Roadtrip → Main Chat
```

This prevents project context from being fragmented across multiple AI conversations.

---

It supports:

* feature detection

### 📈 Momentum & Next-Step Planning

### 🧪 Learning by Doing

Roadtrip is intentionally built around iterative learning.

It supports workflows where developers:

1. Build something.
2. Review what happened.
3. Extract learnings.
4. Improve understanding.
5. Build something slightly more complex.

This is why Roadtrip evolves toward features like:

* Learning Tracker
* Feature Blueprint / App Composer
* Fork-based project relationships
* Architecture onboarding
* Migration planning
* Sprint review systems

Roadtrip is designed to help users become **more capable with less AI dependence over time**.

---

## 🧱 Architecture

Roadtrip is intentionally lightweight.

Current Roadtrip 1.x:

* Single-file HTML app
* Vanilla JavaScript
* no framework
* no backend
* no build step
* localStorage persistence
* optional GitHub Gist sync
* optional Trello integration

This makes it:

* portable
* inspectable
* AI-friendly
* easy to patch
* easy to iterate

---

## 🛣️ Long-Term Direction

Roadtrip is not only a tool.
It is also part of a learning progression.

Planned direction:

```text
Roadtrip 1.x → Single-File HTML / localStorage
Roadtrip 2.0 → Flask / SQLite / stronger architecture separation
Roadtrip 3.x → Desktop / broader platform thinking
Roadtrip Mobile → long-term optional expansion
```

The purpose is not complexity for its own sake.
The purpose is growing architecture maturity over time.
